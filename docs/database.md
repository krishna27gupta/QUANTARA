# Database Schema Documentation

This document describes the PostgreSQL relational database schemas designed for the **Quantara** swing trading platform.

---

## 💻 Tables Overview

### 1. `users`
Core account records storing authentication credentials.
* `id` (UUID, Primary Key): Unique identifier.
* `email` (VARCHAR, Unique, Indexed): User email.
* `hashed_password` (VARCHAR): Encrypted password hash.
* `is_active` (BOOLEAN): Status toggle.
* `created_at` (TIMESTAMP): Creation date.

### 2. `profiles`
User profile details linking back to user accounts.
* `id` (UUID, Primary Key): Profile identifier.
* `user_id` (UUID, ForeignKey -> `users.id`): References the user.
* `full_name` (VARCHAR): Full name.
* `avatar_url` (VARCHAR): Profile photo URL.
* `subscription_status` (VARCHAR): Level status (free, premium, pro).
* `updated_at` (TIMESTAMP): Date updated.

### 3. `stocks`
Securities metadata mapped to NIFTY 50 constraints.
* `id` (UUID, Primary Key): Stock index key.
* `ticker` (VARCHAR, Unique, Indexed): Symbol (e.g., RELIANCE, TCS).
* `company_name` (VARCHAR): Official corporate name.
* `exchange` (VARCHAR): Trading exchange (defaults to NSE).
* `is_active` (BOOLEAN): Flag checking if stock is monitored.

### 4. `market_data`
OHLC pricing logs for tracked tickers.
* `id` (UUID, Primary Key): Market record key.
* `stock_id` (UUID, ForeignKey -> `stocks.id`): Target security.
* `open_price` (DECIMAL): Market open value.
* `close_price` (DECIMAL): Market close value.
* `high_price` (DECIMAL): Market session high.
* `low_price` (DECIMAL): Market session low.
* `volume` (INTEGER): Session volume count.
* `timestamp` (TIMESTAMP, Indexed): Time of pricing logs.

### 5. `predictions`
ML forecasting metrics.
* `id` (UUID, Primary Key): Model prediction key.
* `stock_id` (UUID, ForeignKey -> `stocks.id`): Target security.
* `prediction_date` (TIMESTAMP): Target evaluation date.
* `target_price` (DECIMAL): Predicted asset price target.
* `confidence_score` (DECIMAL): Confidence percentage (0-100).
* `direction` (VARCHAR): Bullish, Bearish, or Neutral.
* `created_at` (TIMESTAMP): Generation date.

### 6. `recommendations`
Consolidated trade buy/sell signals generated by ML/AI.
* `id` (UUID, Primary Key): Signal record identifier.
* `stock_id` (UUID, ForeignKey -> `stocks.id`): Target stock.
* `recommendation_type` (VARCHAR): BUY, SELL, HOLD.
* `reason` (TEXT): Strategy rationale logic explanation.
* `score` (DECIMAL): Relative rank score.
* `created_at` (TIMESTAMP): Generation date.

### 7. `portfolios`
Grouping of assets owned by a user.
* `id` (UUID, Primary Key): Portfolio identifier.
* `user_id` (UUID, ForeignKey -> `users.id`): Owner reference.
* `name` (VARCHAR): Profile portfolio name label.
* `cash_balance` (DECIMAL): Available cash currency balance.
* `created_at` (TIMESTAMP): Creation date.

### 8. `holdings`
Stock share positions contained inside user portfolios.
* `id` (UUID, Primary Key): Holding record identifier.
* `portfolio_id` (UUID, ForeignKey -> `portfolios.id`): Parent portfolio.
* `stock_id` (UUID, ForeignKey -> `stocks.id`): Held security.
* `shares` (DECIMAL): Quantity of shares.
* `average_buy_price` (DECIMAL): Entry purchase cost basis.
* `updated_at` (TIMESTAMP): Date updated.

### 9. `paper_trades`
Mock swing trading transaction histories.
* `id` (UUID, Primary Key): Trade log key.
* `user_id` (UUID, ForeignKey -> `users.id`): Trader reference.
* `stock_id` (UUID, ForeignKey -> `stocks.id`): Transacted stock.
* `action` (VARCHAR): BUY or SELL.
* `quantity` (DECIMAL): Shares count.
* `execution_price` (DECIMAL): Trade fill value.
* `timestamp` (TIMESTAMP): Trade timestamp.

### 10. `subscriptions`
Billing tiers mapping paid user plans.
* `id` (UUID, Primary Key): Subscription record key.
* `user_id` (UUID, ForeignKey -> `users.id`): Subscriber.
* `plan_name` (VARCHAR): Starter, Pro, Enterprise.
* `price` (DECIMAL): Recurring cost value.
* `status` (VARCHAR): active, inactive.
* `expires_at` (TIMESTAMP): Plan expiration limit date.
* `created_at` (TIMESTAMP): Date signed.

### 11. `conversations`
Chat history logs referencing AI trading sessions.
* `id` (UUID, Primary Key): Chat identifier.
* `user_id` (UUID, ForeignKey -> `users.id`): User.
* `title` (VARCHAR): Conversational topic title.
* `created_at` (TIMESTAMP): Start date.

### 12. `memories`
LLM contextual memories recording user preferences.
* `id` (UUID, Primary Key): Memory key.
* `user_id` (UUID, ForeignKey -> `users.id`): User.
* `conversation_id` (UUID, ForeignKey -> `conversations.id`): Chat context link.
* `key` (VARCHAR): Memory variable name.
* `value` (TEXT): Extracted user insight.
* `created_at` (TIMESTAMP): Generation date.
